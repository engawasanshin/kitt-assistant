# K.I.T.T. Architecture

> このドキュメントは Claude Code・KITT・塩川さんの三者が共有するアーキテクチャ設計書。
> 実装・会話・記憶の全ての判断基準となる。

---

## 核心ビジョン

**「ウインドウはKITTに従属する」**

- KITTがマスターインターフェース。塩川さんとの会話が唯一の正式なやり取り
- チャート・ウォッチリスト・各種表示パネルは**視覚補助サブウインドウ**
- サブウインドウはKITTが「見せると判断した情報」を視覚化したもの
- KITTはサブウインドウの内容も把握し、塩川さんに言葉で説明できる

```
塩川さん
    ↕ 会話
 [K.I.T.T.]  ← マスターインターフェース（Gemini + システムプロンプト）
    ↓ 指示・データ参照
  ┌──────────────────────────────┐
  │  サブウインドウ群（従属）      │
  │  ・ウォッチリスト              │
  │  ・テクニカルチャート          │
  │  ・日々の記録エリア            │
  │  ・ダッシュボード（時刻・挨拶）│
  └──────────────────────────────┘
    ↑ 状態をリアルタイム注入
 getAppStateContext()
```

## コンポーネント役割

### KITT（Gemini ベース）
- **責任**: 塩川さんとの会話、判断、提案、説明
- **できること**: 会話・記憶の活用・サブウインドウ内容の解釈・能動的な報告
- **できないこと**: localStorage への直接書き込み・コードの変更・GitHub push

### サブウインドウ群（JavaScript 制御）
| コンポーネント | 役割 | KITTへの注入 |
|---|---|---|
| ウォッチリスト | 株価の一覧表示・スパークライン | `getAppStateContext()` |
| テクニカルチャート | 詳細チャート・MA・BB | `_lastChartState` → context |
| 日々の記録エリア | 今日の気分・TODO | `getDailyLogContext()` |
| ダッシュボード | 挨拶・時刻・コメント | `getCurrentTimeContext()` |

### Claude Code
- **責任**: コード実装・GitHub push・ファイル管理・設計
- **実装ルール**: `CLAUDE.md` 参照

## データフロー原則

### 1. サブウインドウ → KITT（注入）
サブウインドウが表示しているデータは `getAppStateContext()` 経由でKITTのシステムプロンプトに**毎回注入**される。
KITTは注入されたデータを「📊 アプリの実データ」として参照できる。

```
[ウォッチリスト表示中] → ctx += "INPEX: ¥2,340 (+1.2%)" → KITT が知っている
[チャート表示中]       → ctx += "最後のチャート: INPEX テクニカル分析済み" → KITT が知っている
[今日のログ記録済み]   → ctx += "今日の気分: よかった(+1)" → KITT が知っている
```

### 2. 重複分析の禁止
KITTがサブウインドウのデータを受け取っている場合、**同じ分析を会話で再実行してはいけない**。

例:
- ✅ 「チャートを見ると、INPEXは5日MAを上回っています（📊）」← 注入データを参照
- ❌ 「INPEXは上昇トレンドと推測されます」← 注入データがないのに推測
- ❌ アスキーチャートや架空数値の生成

### 3. KITT が「見ている」状態の定義
以下の場合、KITTはサブウインドウの内容を把握している:
- `getAppStateContext()` でウォッチリストデータが注入されている
- `_lastChartState` にチャートデータが存在し context に含まれている
- `getDailyLogContext()` で今日のログが注入されている

## ファイル構成

```
KITTAssistant/
├── index.html          # アプリ本体（全機能）
├── kitt_memory.md      # KITTの長期記憶（Claude Codeが管理）
├── ARCHITECTURE.md     # このファイル（設計原則）
```
