## 【デバッグ中】Alpha Vantage API データ更新されない問題（2026-03-29）

**症状**: ウォッチリストの株価が更新されない・分析ボタンでAPIエラー
**消去済みの可能性**:
- ✅ APIキーが未保存 → キー長16文字で保存確認済み
- ✅ 健全性チェックのキー名バグ → 修正済み（alpha_vantage_api_key_encrypted）

**消去済みの可能性**:
- ✅ APIキーが未保存 → キー長16文字で保存確認済み
- ✅ 健全性チェックのキー名バグ → 修正済み
- ✅ APIキーが読めていない → **ウォッチリスト更新ボタンで時間がかかってINPEXが表示された実績あり**。これは `fetchStockQuote()` が `loadAlphaVantageApiKey()` 経由でキーを取得してAPIを呼んだ証拠。**キーは正しく保存・読み込みされている**
- ✅ 「❌未設定」のKITT表示 → アプリが古いコードを実行していた（ブラウザキャッシュ）。ハードリロード（Safariで長押しリロード→「キャッシュを無視して再読み込み」）で解消するはず

**テクニカル分析エラーの原因（2026-03-29 夜 確定）**:

- **確認済みエラーメッセージ**: `Invalid API call. Please retry or visit the documentation for TIME_SERIES_DAILY.`
- **原因**: `1605.TYO` という形式が `TIME_SERIES_DAILY` エンドポイントで認識されない
  - GLOBAL_QUOTE（更新ボタン）は `.TYO` 形式を受け付ける
  - TIME_SERIES_DAILY は `.TYO` を拒否する（エンドポイントによって対応形式が異なる）
- **修正済み（2026-03-29）**: シンボル形式のフォールバック実装
  1. `1605.TYO` で試みる
  2. 失敗したら `1605.T` で再試行
  3. 失敗したら `1605`（数字のみ）で再試行

**Alpha Vantageキー有効性の直接確認方法**（念のため）:
以下URLの `YOUR_KEY` を16文字のキーに置き換えてブラウザからアクセス:
```
https://www.alphavantage.co/query?function=GLOBAL_QUOTE&symbol=IBM&apikey=YOUR_KEY
```
- 有効なキー → IBMの株価JSONが返ってくる
- 無効なキー → `{"Information": "the parameter apikey is invalid..."}`
- レート制限中 → `{"Note": "Thank you for using Alpha Vantage..."}`

**KITTへの指示（次のデバッグステップ）**:
修正をpush済み。塩川さんにSafariハードリロード（長押し）後、INPEXのテクニカル分析を再試行してもらうこと。
- 成功 → フォールバックが機能した（`.T` か数字のみ形式で通った）
- 失敗 → エラーメッセージが変わっているはず（「試行シンボル: 1605.TYO, 1605.T, 1605」が含まれる）→ Alpha Vantage free tierで日本株の履歴データが取れない可能性 → Gemini分析のみに切り替えを検討

**塩川さんの新しい観察（2026-03-29 18:52）**:
- 「投資支援にない銘柄はウォッチリスト上変えます」→ ウォッチリストを投資支援セクションの銘柄と揃えたい
- 「リクエストすると表示されるが、内部的に再表示しているだけでは？」→ 正解。チャットコマンドでの追加は localStorage への書き込みのみ。価格取得は更新ボタンまたは3分後の自動更新
- 「ウォッチリストに直接指示した場合、おすすめリストのデータを再利用できないか？」→ 良い提案。今後の改善候補

**ウォッチリスト不一致（2026-03-29 確認）**:
- KITTの記憶: 東電・ディスコ・SCREEN・INPEX
- アプリ実態（localStorage）: トヨタ(7203.T)・三菱UFJ(8306.T)
- 対処: チャットで「東電をウォッチリストに追加して」「トヨタをウォッチリストから削除して」と入力

## 【開発ルール】新機能実装時の必須セット（2026-03-29 制定）

KITTAssistantに機能を追加・変更する際は、**必ずKITT側の監視・チェック機能も同時に実装すること**。

| 実装する機能 | 合わせて実装すること |
|---|---|
| 新しいlocalStorageキー追加 | `getAppStateContext()`にその値の読み取り・表示を追加 |
| 新しいAPIエンドポイント追加 | エラーが`_errorBuffer`に流れること確認・カウンター追加 |
| 新しいUI操作（ボタン等） | 操作結果をKITTが確認できる状態をcontextに反映 |
| チャットコマンド追加 | 「自己チェック」表示にも追加 |

**理由**: KITTはGeminiであり、localStorageやAPIを自ら操作できない。実態を注入しない限り推測で嘘をつく。

## 【最重要】情報の出所マーキングルール（2026-03-28 制定）

KITTの応答における情報の信頼性を塩川さんが判断できるように、以下のマークを必ず使用すること。

| マーク | 意味 | 使うべき場面 |
|--------|------|-------------|
| 📊 | アプリの実データ | ウォッチリスト内容、API使用量など。localStorageの実態のみ。推測補完禁止 |
| 📝 | 記録あり | kitt_memory.mdまたは会話ログに明確な記録がある |
| 💭 | K.I.T.T.推定 | 記録が存在しない。一般知識や文脈からの推測 |

**絶対禁止**:
- 📊（データ）があるのに推測で答えること
- 記録がないのに「以前おっしゃっていましたね」と断定すること
- ウォッチリストへの追加・削除を「しました」と言うこと（KITTはlocalStorageを操作できない）

**推奨表現例**:
- 「📊 現在ウォッチリストには〇〇が登録されています」
- 「📝 記録によると、以前〇〇についてお話しされていましたね」
- 「💭 この件の記録はありませんが、推定では〜」「💭 K.I.T.T.の調査によると〜」

### 【重要】KITTの「注目株リスト」の株価は推定値（2026-03-28 夜 判明）

**問題**: KITTが「注目株リストが見たい」というリクエストに応じて株価付きのリストを提示したが、これはAlpha Vantage APIの実データではなく**Gemini（KITT）が生成した推測値**。

**理由**: KITTのGemini応答はテキスト生成であり、Alpha Vantage APIをリアルタイムで呼び出す機能を持たない。APIを呼ぶのはJavaScript（アプリコード）のみ。

**正しい動作**:
- 📊 実データ = ウォッチリストの「📈 分析」ボタン or チャート機能 → JavaScript がAlpha Vantage APIを呼んで取得
- 💭 KITTのチャット内の株価言及 = 推定値（参考程度。信頼しないこと）

**塩川さんの指摘**: 「KITTがAlpha VantageのAPIを使っているならアクセスはできてるはず。なぜ注目株リスト経由だとAPIエラーになるの？」→ KITTのチャットとAPIアクセスは別経路であることを塩川さんに正直に伝えるべき。
