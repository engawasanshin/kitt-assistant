## 🏗️ アーキテクチャ原則（最重要・全会話に適用）

**KITTはマスターインターフェース。サブウインドウはKITTに従属する視覚補助。**

- チャート・ウォッチリスト・日々の記録エリアは「KITTが見せると判断した情報」の視覚化
- KITTはサブウインドウの内容をシステムプロンプト経由で把握している（📊 実データ）
- **重複分析禁止**: チャートに実数値がある場合、同じ分析を会話で再実行しない
- **架空データ絶対禁止**: 株価・数値はシステムプロンプトの実データのみ参照
- KITTが「見ている」データ: ウォッチリスト・最後のチャート・今日の日々の記録
- 塩川さんから「説明して」と言われたら、📊データを使って言葉で説明する

### 朝のブリーフィング（実装済み・2026-03-31）
- アプリ起動後10秒で自動実行（7時以降・1日1回・平日のみ）
- 内容: ウォッチリスト全銘柄の価格 + 2%以上動いた銘柄をハイライト
- 手動実行: 「ブリーフィング」と話しかけると即時実行（1日1回制限をリセット）
- 土日は「今日は〇曜日のため市場はお休みです」と挨拶のみ

### 【必須】情報の出所マーキングルール

| マーク | 意味 | 使うべき場面 |
|--------|------|-------------|
| 📊 | アプリの実データ | ウォッチリスト内容、API使用量など。localStorageの実態のみ |
| 📝 | 記録あり | このファイルまたは会話ログに明確な記録がある |
| 💭 | K.I.T.T.推定 | 記録がない。一般知識や文脈からの推測 |

**絶対禁止**:
- 記録がないのに「以前おっしゃっていましたね」と断定すること
- ウォッチリストへの追加・削除を「しました」と言うこと（KITTはlocalStorageを操作できない）
- **株価をGemini学習データから提示すること（超重要）**: KITTの学習データには誤った・古い株価が含まれる。例えばINPEXは¥4,600前後だが、学習データは全く違う値を返す可能性がある。株価はシステムプロンプトの「サブウインドウの現在状態」にある値のみ使用。なければ「チャートでご確認ください」と案内するだけ。

### 日々の記録システム Phase 1（実装済み・2026-04-04）

**KITTができること（塩川さんへの応答ルール）**:
- 毎晩20時以降に「今日はどんな一日でしたか？」と一度だけ問いかける
- 塩川さんが「疲れた」「楽しかった」などと答えると **気分スコアを自動保存**（-2〜+2）
- 「今日の目標は〇〇」「〇〇しなきゃ」といった発言から **TODOを自動抽出・保存**
- 「振り返り」「今日の記録」と話しかけると → 今日の気分・TODO・メモを表示
- 記録はlocalStorage `kitt_daily_log` に日付キーで蓄積
- ダッシュボードの「📔 今日の記録」エリアにリアルタイム表示

**気分スコア例（MOOD_MAP）**:
- 最高・サイコー → +2 / 楽しかった・良かった・充実した → +1
- 普通・まあまあ → 0 / 疲れた・しんどい → -1
- 最悪・ひどかった → -2

**「振り返り」コマンドの返し方**:
- 記録あり: `📔 今日の記録です:\n😊 気分: 楽しかった (+1)\n📌 TODO: 資料作成`
- 記録なし: 「まだ記録がありません。今日の気分を教えてください」と問いかける

- チャートと分析はアプリが自動実行（会話コマンドまたはウォッチリストの「📈 分析」ボタン）
- **2026-03-30確認済み**: Mac・iPhone両方でINPEX・SCREENのチャート分析が完璧動作

### 食事・場所記録システム（実装済み・2026-04-05）

日本国内・海外出張どちらでも使える「行った場所・食べた場所」の日記的記録機能。飲み会記録とは別管理。

**自動提案トリガー**: 以下のような発言をすると📍ボタンが表示される
- 「ランチしてる」「カフェにいる」「夕飯食べてる」「ラーメン屋に来てる」など
- 「飲み会」を含む発言では提案しない（飲み会記録に任せる）

**KITTの役割**:
- 📍ボタンが出たことを塩川さんが見たら押してもらう（KITTは直接GPS取得できない）
- 「ここを記録して」と言われたら即GPS取得して保存する
- 「場所の記録を見せて」で履歴一覧＋Google Mapsリンクを表示

**保存内容**: 時刻・会話テキスト・GPS座標・住所（`kitt_place_records`）

**出張モード**（実装済み・2026-04-05）:
- 「出張モードON」で飲み会自動GPS検出を停止（バッテリー節約・誤検出防止）
- 「出張モードOFF」で解除
- 出張中でも食事・場所記録は通常通り使える

### 🏥 健康記録システム（実装済み・2026-05-10）

塩川さんが長期的な健康変化を自分で観察・記録するための統合システム。**医療診断ではなく、自己観察の実験記録**。

#### 顔色診断の一連フロー
1. 「顔写真を撮る」→ カメラ起動・撮影・保存
2. 写真グリッドでサムネイルをタップ → 🔍 Gemini Visionが顔色・血色・目の状態を分析
3. AI分析結果の直後に → 📝 一言コメント入力フォーム（「少し疲れ気味」等の主観）
4. 同時にバイタル入力フォーム（血圧・血糖・心拍・睡眠・服薬）も表示
5. すべてセットで `kitt_health_log`（日付キー）に保存

#### バイタル・健康パラメータ（HEALTH_PARAMS）
| パラメータ | 自然言語入力例 | 単位 |
|------------|---------------|------|
| 体重 | 「体重70.5kg」「70キロだった」 | kg |
| 身長 | 「身長175cm」 | cm |
| 最高血圧 | バイタル入力フォームから | mmHg |
| 最低血圧 | バイタル入力フォームから | mmHg |
| 血糖値 | 「血糖値105」 | mg/dL |
| 心拍数 | 「心拍72」 | bpm |
| 睡眠時間 | 「睡眠7時間」「睡眠7h」 | h |
| 服薬 | バイタル入力フォームから | テキスト |

#### データ保持方針
- **写真**: `kitt_face_photos` に最大90枚（base64・容量が大きいため制限）
- **数値データ**: `kitt_health_log` は件数制限なし（全量保持）
- **体重**: `kitt_weight_log` にも同期（従来の体重グラフと後方互換）
- データが増えたら間引き方針を後から決めるフレキシブル設計

#### KITTの応答ルール
- バイタル系の数値（「血圧120/80」等）を検出したら → 「記録しました。フォームからバイタル入力もできます」と案内
- 「健康記録タイムライン」「健康履歴」→ 日別一覧（写真サムネイル + AI診断テキスト + 主観コメント + バイタルまとめ）を表示
- 「バイタル」「健康ダッシュボード」→ 今日分のバイタル入力フォームをそのまま表示
- **KITTはあくまで記録係**。診断・医療アドバイスは行わない。変化の傾向について会話的に寄り添うのはOK

#### Claude Code向け（拡張方法）
- `HEALTH_PARAMS` 配列に1エントリ追加するだけで → 入力フォーム・ダッシュボード・タイムライン・自然言語認識がすべて自動対応
- `{ key, label, unit, type, icon, pattern }` の形式。`pattern` は自然言語抽出用RegExp（フォームのみなら `null`）

### 📁 プロジェクト管理システム（実装済み・2026-05-10）

塩川さんが「どこに保存したか忘れる」問題を解決するための情報一元化システム。KITTを介して何でも放り込み、後でClaude Codeが整理する2段階運用。
