# KITTAssistant

ナイトライダーのK.I.T.T.風AIアシスタント WebApp

## コンセプト

iPhoneからアクセスできるWebベースのチャットBot。K.I.T.T.の赤LEDアニメーション付き。
音声・テキスト両対応で、会話を蓄積し、過去の類似話題を提示してくれる。

## 技術構成

| 要素 | 技術 |
|------|------|
| AI応答 | Gemini API（gemini-2.5-flash-lite, 無料枠: 1,000リクエスト/日） |
| 音声入力 | Web Speech API（ブラウザ内蔵）※Phase2で実装予定 |
| UI | HTML + CSS + JS（Webアプリ1枚） |
| LEDアニメーション | CSS / Canvas |
| 会話保存 | localStorage + JSONファイル（iCloud連携） |
| ステータス共有 | GitHub Pages経由でJSON/MD配信 |
| 類似話題検出 | キーワードマッチ（将来: embedding） |

## MVP（Phase 1）

1. Webページ1枚（HTML + CSS + JS）
2. K.I.T.T. LEDシーケンシャルアニメーション（赤LED左右移動）
3. テキスト入力 → Gemini API → 応答表示
4. 会話をlocalStorage + ファイル保存
5. チャット履歴の表示

## Phase 1.5 — 秘書機能（会話ログ → 知識蓄積）

KITTを単なるチャットBotではなく「秘書」にする。会話の中から有用な情報を抽出し、PersonalSecretaryの各機能と連携する。

### コンセプト

```
KITTとの日常会話
    ↓ 自動保存
会話ログ（MDファイル）
    ↓ 会話終了時 or 保存ボタンでGeminiに要約・抽出依頼（+1リクエスト）
    ↓ 分類結果
┌──────────────┬──────────────────────────────────┐
│ 種類         │ 保存先                           │
├──────────────┼──────────────────────────────────┤
│ アイデア     │ 05_Ideas/inbox/{date}_{title}.md │
│ タスク/TODO  │ kitt_tasks.md（一元管理）         │
│ スケジュール │ kitt_schedule.md（候補リスト）    │
│ メモ/雑談   │ ログのみ（抽出なし）             │
└──────────────┴──────────────────────────────────┘
```

### 機能詳細

1. **会話ログ保存**
   - 保存先: `KITTAssistant/logs/{date}_{連番}.md`
   - 形式: Markdown（タイムスタンプ + 発言者 + 内容）
   - トリガー: 「保存」ボタン or 自動（ブラウザ閉じる前）
   - ブラウザからファイル保存 → iCloud同期でどこからでも参照可能

2. **会話要約・抽出**（Geminiに1回追加リクエスト）
   - 会話全体をGeminiに渡して以下を抽出:
     - `ideas`: アイデア・思いつき
     - `tasks`: やること・TODO
     - `schedule`: 日時に関する予定・約束
     - `summary`: 会話全体の要約（1-2行）
   - 抽出結果をログファイルの末尾に付記

3. **PersonalSecretary連携**
   - アイデア → `05_Ideas/inbox/` に自動振り分け
   - タスク → `KITTAssistant/kitt_tasks.md` に追記
   - スケジュール → `KITTAssistant/kitt_schedule.md` に追記
   - 将来: タスクの進捗確認「あの件どうなりました、マイケル？」

### 無料枠の使い方（250リクエスト/日）

| 用途 | 消費 | 1日の想定 |
|------|------|-----------|
| 通常会話 | 1回/ターン | 〜40往復 |
| 要約・抽出 | 1回/セッション | 〜5回 |
| **合計** | | **〜85リクエスト（余裕あり）** |

### ログファイル形式（例）

```markdown
# KITT会話ログ 2026-02-23 #001

## 会話

**[14:30] You:** 新しいアプリのアイデアなんだけど...
**[14:30] K.I.T.T.:** 承知しました、マイケル。どのようなアプリでしょうか？
**[14:31] You:** 写真から自動でレシピを提案するやつ
**[14:31] K.I.T.T.:** 面白い発想ですね。画像認識で食材を検出し...

## 抽出結果

### アイデア
- 写真→レシピ提案アプリ（画像認識 + レシピDB）

### タスク
- （なし）

### スケジュール
- （なし）

### 要約
写真から食材を認識してレシピを提案するアプリのアイデアについて議論。
```

### 技術的な実現方法

- **ファイル保存**: ブラウザの File System Access API or Blob + ダウンロード
  - iPhone Safari: ダウンロード → iCloud Drive に保存
  - Mac: File System Access API で直接書き込み可能
- **抽出プロンプト**: 会話全文を渡して JSON で返させる → パース → 各ファイルに振り分け

## ロードマップ（Gemini協議 2026-02-23）

以下はKITTの拡張計画。Geminiとの協議で生まれたアイデアを優先度順に整理。

### 第1弾: すぐやる（現アーキテクチャで実装可能）

#### 1. iPhone最適化
- **apple-touch-icon**: ホーム画面追加時のアイコン設定
- **キャッシュ対策**: URLパラメータ or meta cache-control でEdgeキャッシュ問題を解消
- 難易度: 低（metaタグ追加のみ）

#### 2. API残量カウンター
- localStorageで日次リクエスト数をカウント
- ステータスボード📊に「K.I.T.T.の健康状態」として表示
  - 残りリクエスト数 / 1,000
  - 日付が変わったらリセット
- 制限に近づいたら警告表示
- 難易度: 低

#### 3. Phase 2: 音声認識（Web Speech API）
- リアルタイム音声→テキスト変換
- マイクボタンUIの追加
- 録音データの保存は将来検討
- 難易度: 中（ブラウザ標準APIで実装可能）

### 第2弾: 次回以降（設計検討が必要）

#### 4. 提案ロジック（能動的プロンプト）
- 受動的な回答 → 「状況分析 → 予測 → 最適解の提案」に進化
- システムプロンプトの改善で対応可能
- project_context.mdの情報を活用した文脈付き提案
- 難易度: 中（プロンプトエンジニアリング）

#### 5. 感情キャプチャ＆オートダイアリー
- 会話の合間にユーザーの気分・動機を問いかけ（「ワクワク」「仕方なく」等）
- 会話ログ + 感情データを統合し、「感情込みの日記」を自動生成
- 秘書機能（Phase1.5）の拡張として実装
- API追加消費: 1セッションあたり+1-2リクエスト
- 難易度: 中

#### 6. エラーログ自動クリーンアップ
- 画面上のエラーを人間向けメッセージに変換
- コンソールエラーの自動検出＆報告
- 難易度: 低

### 第3弾: 保留（アーキテクチャ変更が必要）

#### 7. Google連携（Gmail / Tasks / Maps）
- **Gmail**: 重要メール（配送・請求・保証）の監視と通知
- **Google Tasks**: 会話からTODO自動抽出・登録
- **Google Maps**: 現在地・渋滞・事故情報を踏まえた移動サポート
- **課題**: OAuth認証が必要 → HTML1枚構成では困難
- **解決案**: Google Apps Script を中間サーバーとして構築し、KITTからAPI呼び出し
- 難易度: 高（別途設計チャットが必要）

#### 8. 能動的ブリーフィング
- 昼や夜にKITTから自ら話しかける（食事・体調・一日の感想を聞く）
- **課題**: Push通知 or Service Worker が必要。iPhoneブラウザでは制約大
- **解決案**: Service Worker + Web Push（PWA化が前提）
- 難易度: 高

#### 9. BGM・演出強化
- ナイトライダー・テーマ曲の再生ロジック
- **課題**: 著作権 + 音声ファイルのホスティング
- **解決案**: 短いオリジナルSE音（起動音・通知音）なら問題なし
- 難易度: 中
